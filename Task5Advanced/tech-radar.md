# Расширенный технический радар системы "Будущее 2.0"

## Легенда статусов

- **Adopt** — использовать активно, рекомендовать для новых проектов
- **Trial** — пробовать в ограниченных проектах, оценивать применимость
- **Assess** — изучать, оценивать целесообразность внедрения
- **Hold** — не использовать для новых проектов, поддерживать только существующие

## Архитектурные паттерны

### Паттерны интеграции

| Паттерн | Статус | Обоснование |
|---------|--------|-------------|
| **Event-Driven Architecture** | Adopt | Основа целевой архитектуры, слабая связанность доменов |
| **CQRS (Command Query Responsibility Segregation)** | Trial | Разделение чтения и записи, оценивать для высоконагруженных доменов |
| **Saga Pattern** | Adopt | Управление распределёнными транзакциями между доменами |
| **API Gateway Pattern** | Adopt | Единая точка входа для внешних клиентов |
| **Anti-Corruption Layer** | Adopt | Интеграция с легаси-системами, изоляция новой архитектуры |
| **Strangler Fig Pattern** | Adopt | Постепенная замена легаси-систем новыми компонентами |

| Паттерн | Статус | Обоснование |

## Приоритеты внедрения

### Высокий приоритет (Adopt)
- Event-Driven Architecture
- Data Mesh
- Apache Kafka
- Kubernetes
- Self-service BI
- Data Lake (Iceberg)
- Потоковая обработка (Flink)

### Средний приоритет (Trial)
- CQRS для критичных доменов

- Multi-tenancy для геораспределения

### Низкий приоритет (Assess)
- Service Mesh
- Data Vault
- ClickHouse / Druid для специфических сценариев

### Вывод из эксплуатации (Hold)
- Lambda Architecture
- Apache Camel (кроме ACL)
- SQL Server 2008
- Power Builder

## Критерии оценки технологий

1. **Зрелость и стабильность** — проверенные решения с активным сообществом
2. **Масштабируемость** — способность расти с бизнесом
3. **Интеграция** — совместимость с существующим стеком
4. **Безопасность** — соответствие требованиям регуляторов
5. **Поддержка** — доступность экспертизы и документации
6. **Стоимость** — TCO включая лицензии и поддержку
7. **Производительность** — соответствие требованиям по latency и throughput
